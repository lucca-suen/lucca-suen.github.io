动态规划思考方式

闫氏DP分析法（北大NOI金牌yxc发明的，用来理解DP的逻辑模型，非常牛）

  • 状态表示：定义表达式来解释问题的集合
    • 集合：具体的含义
    • 属性：常见有三种：最大值、最小值、数量
  • 状态计算：如何把当前集合划分成更小的子集，使得每一个集合都可以算出来（用前面更小的状态表示出来）
    • 不重：同一个状态不能重复出现
    • 不漏：不能漏掉任何一个元素
  • 状态初始化：循环如何开始，初始状态设置

背包问题

给定一个固定大小的背包，给定若干物品，每个物品有自己的体积和价值

一般求在规定背包空间内（不一定要装满背包），能积累的最大价值

01背包

每件物品最多只用一次

  • 状态表示：$f_{i,j}$

    • 集合：考虑前i个物品，空间大小为j时的价值
    • 属性：最大值

  • 状态计算：

    第 $i$ 个物品，容量为 $j$ 时，其取值从以下两个集合而来

    要么不取，其价值等于前 $i-1$ 个物品，大小为 $j$ 的价值，化简后为 $f_{i-1,j}$

    要么取，其价值等于前 $i-1$ 个物品，大小为 $j-v_i$ 的价值，同时还要补上自身的价值 $w_i$ ，化简后为 $f_{i-1,j-v_i} + w_i$

  • 状态初始化：全为0

朴素版

一维滚动数组 空间优化

  • 每次只涉及到 $i$ 与 $i-1$ 层的数组，可以使用滚动数组优化空间复杂度

完全背包

每件物品有无限个

简化为01背包

  • 状态表示：$f_{i,j}$

    • 集合：考虑前i个物品，空间大小为j时的价值
    • 属性：最大值

  • 状态计算：

    第 $i$ 个物品，容量为 $j$ 时，其取值从以下k个集合而来

    • 要么不取，其价值等于前 $i-1$ 个物品，大小为 $j$ 的价值，化简后为 $f_{i-1, j}$
    • 要么取 $1$ 个，其价值等于前 $i-1$ 个物品，大小为 $j-v_i$ 的价值，同时还要补上自身的价值 $w_i$ ，化简后为 $f_{i,j - v_i} + w_i$
    • 要么取 $2$ 个，其价值等于前 $i-1$ 个物品，大小为 $j-2v_i$ 的价值，同时还要补上自身的价值 $2 w_i$ ，化简后为 $f_{i, j - 2 v_i} + 2 w_i$
    • …
    • 要么取 $k$ 个 $(k v_i <= j)$，其价值等于前 $i-1$ 个物品，大小为 $j-kv_i$ 的价值，同时还要补上自身的价值 $kw_i$ ，化简后为 $f_{i, j - k v_i} + k * w_i$

  • 状态初始化：全为0

  • 每次只涉及到 $i$ 与 $i-1$ 层的数组，可以使用滚动数组优化空间复杂度

写法与01背包相同，但是除了枚举物品 $i$ 和容量 $j$ 的循环之外，还要枚举物品个数 $k$

多重背包问题

每个物品个数不同

朴素版

  • 分析思路类比 完全背包方法1

无法使用表达式优化

借鉴完全背包的思路，从表达式入手，不同于完全背包，多重背包每个物品的个数不是无限的。

完全背包每次计算的上限是 $最大个数 = 总容量 / 物品容量$

多重背包的最大个数如果大于上式计算出的最大个数，可以视为完全背包，从而优化。

二进制优化

性质：对于任意一个数 N，我们可以拆成 $1,2,4,...,2^k, N-2^k (2^k < N)$，并可以用这些数凑出从 0 到 N 中的任意一个数

一开始，我们的目的是，在个数为s的物品中找出其最优取值，使得最终方案得到的价值最大。

由于上述性质，0-N 之间任何数字都可以由 $1,2,4,…,2^k, N-2^k$ 构成，我称之为解构数。

那么，可以把0-N之间每个数字，想象成一个位图，每一位代表每一个解构数是否被取。

问题转变成了针对解构数的01背包，也就是寻找如何组合解构数才能构成最优解。

  1. 将每个物品分情况讨论
    1. 如果其最大数量 * 物品占用空间 > 最大空间，表示数量是用不完的，等价于完全背包
    2. 如果其最大数量 * 物品占用空间 < 最大空间，表示数量是可以全部用完的，按照上述性质，将N拆分成若干物品，使用01背包判断每个物品取与不取，计算最大权值

分组背包问题

物品分成不同组，每组只能选一个物品

  • 状态表示：$f_{i,j}$

    • 集合：考虑前 $i$ 组物品，空间大小为 $j$ 时的价值
    • 属性：最大值

  • 状态计算：

    第 $i$ 组物品，容量为 $j$ 时，其取值从以下k个集合而来

    • 要么不取，其价值等于前 $i-1$ 个物品，大小为 $j$ 的价值，化简后为 $f_{i-1, j}$
    • 要么取第 $1$ 个，其价值等于前 $i-1$ 个物品，大小为 $j-v_1$ 的价值，同时还要补上自身的价值 $w_1$ ，化简后为 $f_{i,j - v_1} + w_1$
    • 要么取第 $2$ 个，其价值等于前 $i-1$ 个物品，大小为 $j-v_2$ 的价值，同时还要补上自身的价值 $w_2$ ，化简后为 $f_{i,j - v_2} + w_2$
    • …
    • 要么取第 $k$ 个，其价值等于前 $i-1$ 个物品，大小为 $j-v_k$ 的价值，同时还要补上自身的价值 $w_k$ ，化简后为 $f_{i,j - v_k} + w_k$

  • 状态初始化：全为0

  • 每次只涉及到 $i$ 与 $i-1$ 层的数组，可以使用滚动数组优化空间复杂度
  • 一组中最值的计算放在一起的，所以同一组内物品的遍历需要放在最内层，否则计算的就不是一个集合内的最值
